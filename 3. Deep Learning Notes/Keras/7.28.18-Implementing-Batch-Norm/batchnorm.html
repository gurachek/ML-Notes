Notes on Batch Normalization¶

How it Works¶

Like normalization in regular preprocessing, with batch normalization we seek a zero-centered, unit variance distribution of inputs to our activation functions. During training time we take an activation $x$ and subtract it by the batch mean $\mu_{B}$ to achieve a zero centered distribution.

$\hat{x} = x_i-\mu_{B}$

Then we take $x$ and divide it by the batch variance and a small number to prevent division by zero $\sigma_{B}+\epsilon$

$\hat{x} = \frac{x_i-\mu_{B}}{\sigma_{B}+\epsilon}$

Lastly we send $\hat{x}$ through a linear transformation to scale and shift the output of batch normalization $y_i$

$y_i=\lambda *\hat{x_i}+\beta$

During test time, we do not use batch mean or variance, instead as we train we calculate a moving average and variance to estimate these values for the training population. This simply means retaining the batch means and variances and averaging them together to form an estimate of population mean an variance. These population estimates are used in the batch normalization layers when we propagate a test-sample through the network.

Benefits of Batch Normalization¶

After reading [1] I distill the benefits of batch normalization down to the following.

1. Helps prevent vanishing gradient in networks with saturable nonlinearities (sigmoid, tanh, etc)¶

If $g(x)$ represents the sigmoid function, as $| x |$ increases, $g^\prime(x)$ tends to zero. For a given hidden layer being trained in a neural network, any unit with an exceptionally large absolute output will have a gradient that tends to zero.

So if a large portion of our neurons are outputting exceptionally positive or negative values, then the large area that those neurons are connected to will receive little or no gradient propagation.

With Batch normalization we ensure that the inputs of any activation function do not vary into saturable regions by transforming the distribution of thos inputs to be unit gaussian (zero-centered and unit variance). This should speed up training time. (as gradients are being passed back without vanishing.)

2. Regularizes the model¶

Maybe. Ioffe and Svegeddy make this claim but dont really back it up. Its possible that we are normalizaing the network by constricting the scale of individual neuron outputs, but I need to read further to see if this actually has a normalizing effect.

3. Allows for Higher Learning Rates¶

By preventing issues with vanishing gradient during training, we can afford to set higher learning rates.

Batch normalization also reduces dependence on parameter scale. Large learning rates can increase the scale of layer parameters which cause the gradients to amplify as they are passed back during back propagation. I need to read more about this.

from keras.datasets import cifar100
from keras.utils import np_utils

(x_train, y_train), (x_test, y_test) = cifar100.load_data(label_mode='fine')

#scale and regularize the dataset
x_train = (x_train-np.mean(x_train))
x_test = (x_test - x_test.mean())

x_test = x_test.astype('float32')


#onehot encode the target classes

def conv_block_first(model):
    
    model.add(Conv2D(60,3, padding = "same", input_shape = x_train.shape[1:]))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    
    model.add(Conv2D(60,3, padding = "same"))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    model.add(MaxPooling2D())
    model.add(Dropout(0.15))
    return model



def conv_block(model):
    
    model.add(Conv2D(60,3, padding = "same"))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    model.add(Conv2D(60,3, padding = "same"))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    model.add(MaxPooling2D())
    model.add(Dropout(0.15))
    return model



def conv_block_final(model):
    
    model.add(Conv2D(100,3, padding = "same"))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    
    model.add(Conv2D(5,3, padding = "same"))
    model.add(BatchNormalization())
    model.add(Activation("relu"))
    
    
    model.add(Flatten())
    return model



def fn_block(model):
    model.add(Dense(100, activation = "softmax"))
    return model



def build_model(blocks=3, classes=5):
    model = Sequential()
    
    model = conv_block_first(model)
    
    for block in range(1,blocks-1):
        model = conv_block(model)
        
    model = conv_block_final(model)
    model = fn_block(model)
    
    return model



def compile_model(model, optimizer = "rmsprop", loss = "categorical_crossentropy", metrics = ["accuracy"]): 
    model.compile(optimizer = optimizer,
                 loss = loss,
                 metrics = metrics)
    return model
